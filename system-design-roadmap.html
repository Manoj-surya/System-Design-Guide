PHASE 02 Databases — Deep Dive Week 3–4
SQL Databases
  • ACID properties (Atomicity, Consistency, Isolation, Durability)
  • Indexes, joins, transactions
  • PostgreSQL, MySQL use cases
  • When to prefer SQL (payments, banking)
NoSQL Databases
  • Document DB — MongoDB (flexible schema)
  • Key-Value — Redis, DynamoDB
  • Wide-column — Cassandra (time-series, logs)
  • Graph DB — Neo4j (social networks)
Sharding & Partitioning
  • Horizontal partitioning (sharding)
  • Range-based sharding
  • Hash-based sharding
  • Directory-based sharding

Replication & Migration
  • Leader-follower replication
  • Multi-leader replication
  • Leaderless replication
  • Synchronous vs async replication
  • Zero-downtime migration strategies
CAP Theorem
  • Consistency — every read gets latest write
  • Availability — every request gets a response
  • Partition tolerance — works despite network splits
  • CP systems: HBase, Zookeeper
  • AP systems: Cassandra, CouchDB
Consistency Models
  • Strong / Linearizable consistency
  • Eventual consistency
  • Causal consistency
  • Quorum consistency (read/write quorum)
  • Isolation levels: Read Uncommitted → Serializable

PHASE 05 Low Level Design — OOP & Patterns Week 10–12
OOP Pillars
  • Encapsulation — hide internal state
  • Abstraction — expose only what's needed
  • Inheritance — reuse through parent-child
  • Polymorphism — same interface, different behaviour
SOLID Principles
  • S — Single Responsibility Principle
  • O — Open/Closed Principle
  • L — Liskov Substitution Principle
  • I — Interface Segregation Principle
  • D — Dependency Inversion Principle
Creational Design Patterns
  • Singleton — one instance globally
  • Factory Method — delegate creation to subclass
  • Abstract Factory — families of objects
  • Builder — step-by-step object construction
  • Prototype — clone existing objects
Structural Design Patterns
  • Adapter — convert one interface to another
  • Decorator — add behaviour dynamically
  • Facade — simplified interface to complex system
  • Proxy — control access to an object
  • Composite — tree structures of objects
  • Bridge — decouple abstraction from implementation
Behavioural Design Patterns
  • Observer — notify subscribers on state change
  • Strategy — swap algorithms at runtime
  • Command — encapsulate requests as objects
  • State — change behaviour based on state
  • Template Method — define skeleton, override steps
  • Iterator — traverse collections uniformly
  • Chain of Responsibility — pass request along chain
Concurrency & Thread Safety
  • Race conditions & how to prevent them
  • Mutex, Semaphore, Monitor locks
  • Deadlock, livelock, starvation
  • Producer-Consumer pattern
  • Thread pool pattern
  • Immutability as a safety strategy
API Design
  • RESTful API principles & conventions
  • Request/response object modelling
  • API versioning strategies (URL, header)
  • Idempotency (PUT, DELETE)
  • Error handling & status codes
  • Pagination (cursor vs offset)
UML Diagrams
  • Class diagrams — relationships & multiplicity
  • Sequence diagrams — time-ordered interactions
  • Component diagrams — system structure
  • Use case diagrams — actor-system interaction
Clean Code Principles
  • DRY — Don't Repeat Yourself
  • KISS — Keep It Simple
  • YAGNI — You Aren't Gonna Need It
  • Avoiding God classes
  • Meaningful naming & small functions
